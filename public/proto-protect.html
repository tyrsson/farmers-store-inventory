Prototype Notes
  • For single-method routes (all 14 shown here), the privilege badge is fixed and read-only — derived from the HTTP method via METHOD_PRIVILEGE_MAP. There is nothing to choose.
  • For multi-method routes (e.g. ['GET', 'POST']), the accordion body shows privilege checkboxes instead of a fixed badge — one per unique privilege derived from the allowed methods. Only Read is pre-checked; additional privileges must be explicitly opted into.
  • The Starting Role(s) select is multi-select. Selecting multiple roles creates one allow rule per role. The form posts role[] as an array.
  • Clicking Protect will: (1) create the ACL resource DB row, (2) insert one privilege DB row per unique privilege derived from the route's allowed methods, (3) create an allow rule for each selected role + the derived privilege(s).
  • admin.acl.* routes appear here because they are not yet seeded as protected resources. Once seeded they disappear from this list.
